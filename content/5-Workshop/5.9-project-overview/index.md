## Project Introduction

AWS BILLO is a digital wallet application combined with QR table ordering and QR payment, built on AWS serverless services (Cognito, API Gateway, Lambda, DynamoDB, S3, CloudWatch) and deployed using AWS SAM/CloudFormation.

The application has two frontends, currently hosted on AWS Amplify:

```text
Flutter app   → https://dev.d28z1hw6wfvjzy.amplifyapp.com (Customer, Merchant)
Admin Web     → https://dev.d3k8atm3w5sdj3.amplifyapp.com (Admin)
```

AWS BILLO supports three main roles:

```text
Customer  → register a wallet, transfer money, order and pay via table QR
Merchant  → manage store, products, tables, and receive payments
Admin     → approve business applications, manage users, monitor transactions
```

All data and business flows below are taken from the same development environment deployed in the earlier sections:

```text

Cognito User Pool ID: ap-southeast-1_AKc39KB4L
DynamoDB Main Table: wallet-app-main-dev
```

## 1. Customer Features

### 1.1. Sign Up / Sign In

Customer signs up with phone number and password, then verifies via OTP sent by Cognito/SNS.

![alt text](image-24.png)

Related services: Amazon Cognito, Amazon SNS.

### 1.2. Set Transaction PIN

After signing in, the Customer sets a 6-digit PIN used to confirm transfers and payments.

![alt text](image-23.png)

Related services: Amazon Cognito (profile), DynamoDB Main Table.

### 1.3. Wallet and Transaction History

Customer views wallet balance, recent transactions, and transaction details.

![alt text](image-22.png)

Related services: DynamoDB Main Table (wallet, transaction).

### 1.4. Transfer Money

Customer transfers money to another user by phone number, confirmed with the transaction PIN.

![alt text](image-21.png)

Related services: DynamoDB Idempotency Table (prevents duplicate transactions).

### 1.5. Scan Table QR and Order

Customer scans the store's table QR to open the menu, select items, and submit an order.

![alt text](image-20.png)

Related services: DynamoDB Main Table (table, order).

### 1.6. Pay a QR Bill

Customer scans the payment QR generated by the Merchant, reviews the bill, and confirms payment with the PIN.

![alt text](bill-qr-order_1783924540406.png)

Related services: DynamoDB (payment session, transaction).

## 2. Merchant Features

### 2.1. Business Registration

Customer submits a business registration application, including a business license image, pending Admin approval.

![alt text](image-19.png)

Related services: Amazon S3 (document upload via pre-signed URL).

### 2.2. Business Workspace

Once approved and after re-logging in, the Merchant can access the Overview, Services, Tables, POS, and Orders tabs.

![alt text](image-18.png)

Related services: Cognito User Group `Merchant`.

### 2.3. Manage Categories and Products/Services

Merchant creates categories and adds products/services with name, price, image, sale status, and discount configuration.

![alt text](image-17.png)

Related services: DynamoDB Main Table, Amazon S3 (product images).

### 2.4. Manage Tables and Table QR

Merchant creates tables; the system generates a QR code for each table to print and place on-site.

![alt text](image-16.png)

Related services: DynamoDB Main Table (table).

### 2.5. Receive Orders and Manage Bills

Merchant views each table's orders in real time and can add, remove, or edit items before payment.

![alt text](image-14.png)

Related services: DynamoDB Main Table (order, bill).

### 2.6. Payment

Merchant generates a payment QR for the Customer to scan, or marks the order as paid in cash.

![alt text](image-15.png)

Related services: DynamoDB (payment session, transaction); the Merchant's wallet is credited.

## 3. Admin Features

Admin Web is currently hosted on AWS Amplify at: `https://dev.d3k8atm3w5sdj3.amplifyapp.com`, connecting to the same API Gateway endpoint as the Flutter app.

### 3.1. Admin Sign In

Sign in with an account belonging to the Cognito `Admin` group.

![alt text](image-9.png)

Related services: Amazon Cognito (User Group `Admin`).

### 3.2. Dashboard Overview

Admin sees at a glance the number of pending applications, stores, users, recent transactions, and pending refunds.

![alt text](image-10.png)

Related services: DynamoDB Main Table.

### 3.3. Approve / Reject Merchant Applications

Admin reviews business registration details and decides to approve or reject.

![alt text](image-11.png)

Upon approval, the system automatically adds the user to the `Merchant` group in Cognito and creates a Store record.

### 3.4. Manage Users and Stores

Admin views the user list, locks/unlocks accounts, revokes Merchant permissions, and toggles a store's active status.

![alt text](image-12.png)

Related services: Amazon Cognito (User Group), DynamoDB Main Table.

### 3.5. Transactions and Refunds

Admin views transaction details and approves or rejects refund requests.

![alt text](image-13.png)

Related services: DynamoDB Main Table (transaction, refund).

## Overall Flow

A summary of how the three roles work together, matching the end-to-end demo covered in section 5.7:

```text
Customer signs up & submits business registration
        ↓
Admin approves the Merchant application
        ↓
Merchant sets up store, products, tables, table QR
        ↓
Customer scans table QR, places an order
        ↓
Merchant reviews the bill, generates a payment QR
        ↓
Customer pays using the PIN
        ↓
Admin monitors transactions & handles refunds (if any)
```
