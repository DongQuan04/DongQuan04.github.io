## Project Introduction

AWS BILLO is a digital wallet application combined with QR table ordering and QR payment, built on AWS serverless services (Cognito, API Gateway, Lambda, DynamoDB, S3, CloudWatch) and deployed using AWS SAM/CloudFormation.

The application has two frontends:

```text
Flutter app   → used by Customer and Merchant
Admin Web     → used by Admin
```

AWS BILLO supports three main roles:

```text
Customer  → register a wallet, transfer money, order and pay via table QR
Merchant  → manage store, products, tables, and receive payments
Admin     → approve business applications, manage users, monitor transactions
```

All data and business flows below are taken from the same development environment deployed in the earlier sections:


## 2. Merchant Features

### 2.1. Business Registration

Customer submits a business registration application, including a business license image, pending Admin approval.

![img_6.png](img_6.png)

Related services: Amazon S3 (document upload via pre-signed URL).

### 2.2. Business Workspace

Once approved and after re-logging in, the Merchant can access the Overview, Services, Tables, POS, and Orders tabs.

![img_7.png](img_7.png)

Related services: Cognito User Group `Merchant`.

Merchant creates categories and adds products/services with name, price, image, sale status, and discount configuration.

![img_8.png](img_8.png)

Related services: DynamoDB Main Table, Amazon S3 (product images).

### 2.4. Manage Tables and Table QR

Merchant creates tables; the system generates a QR code for each table to print and place on-site.

![img_9.png](img_9.png)

### 2.5. Receive Orders and Manage Bills

Merchant views each table's orders in real time and can add, remove, or edit items before payment.

![img_10.png](img_10.png)

### 2.6. Payment

Merchant generates a payment QR for the Customer to scan, or marks the order as paid in cash.

## Overall Flow

A summary of how the three roles work together, matching the end-to-end demo covered in section 5.7:

```text
Customer signs up & submits business registration
        ↓
Admin approves the Merchant application
        ↓
Merchant sets up store, products, tables, table QR
        ↓
Customer scans table QR, places an order
        ↓
Merchant reviews the bill, generates a payment QR
        ↓
Customer pays using the PIN
        ↓
Admin monitors transactions & handles refunds (if any)
```
